# 附录C：随机规划 × ML 的交叉点

> 如果你既是 ML 工程师又是运筹优化的人——这里是你应该关注的前沿交叉地带。

---

## 0. 为什么要交叉？

传统的工作流是串行的：ML 负责预测（需求、价格、故障概率），优化负责在预测结果上做决策。

但问题是：**ML 的损失函数（MSE、交叉熵）并不关心下游决策的质量。** 一个预测误差 5% 但导致决策损失 50 万的模型，不如一个预测误差 8% 但只导致决策损失 10 万的模型。

下面介绍五个交叉方向，从最成熟到最前沿。

---

## 1. Predict-then-Optimize —— 最成熟

**结构**：

```
┌──────────┐    预测      ┌──────────┐    决策      ┌──────────┐
│  ML 模型  │ ─────────→  │ 分布/场景  │ ─────────→  │ 优化模型  │
│ (LSTM/GBDT)│            │ (PDF/样本) │             │ (LP/MIP) │
└──────────┘              └──────────┘              └──────────┘
```

**做法**：ML 模型预测需求分布（比如预测均值和方差），然后把预测结果作为随机规划或鲁棒优化的输入。

**例子**：预测下个月每个 SKU 的需求分布 → 输入随机规划的新闻摊贩模型 → 输出最优订货量。

**优点**：模块化、易实现、ML 和优化团队可以独立工作。

**缺点**：ML 模型不知道优化模型在做什么——它只最小化预测误差，不最小化决策损失。

**什么时候用**：团队分工明确、预测和优化是不同部门负责的、或者 ML 模型难以端到端训练。

---

## 2. Decision-Focused Learning（DFL）—— 学术热点

> 核心思想：让 ML 模型直接最小化下游决策损失，而不是预测误差。

**结构**：

```
┌──────────┐    预测      ┌──────────┐    决策损失    ┌──────────────┐
│  ML 模型  │ ─────────→  │ 优化模型  │ ←──────────  │ 决策损失函数  │
│ (可微分)   │             │ (可微分)   │              │ (业务 KPI)   │
└──────────┘              └──────────┘              └──────────────┘
         ↑_________________反向传播__________________↓
```

**关键方法**：

| 方法 | 作者/年份 | 核心思路 |
|------|---------|---------|
| SPO (Smart Predict-then-Optimize) | Elmachtoub & Grigas, 2021 | 定义一个凸的 SPO loss，替代 MSE |
| DFL (Differentiable Optimization) | Agrawal et al., 2019 | 把优化层变成可微分层，嵌入神经网络 |
| I-MLE (Implicit MLE) | Niepert et al., 2021 | 用隐式最大似然估计把组合优化嵌入 ML |

**生活类比**：你教一个新人做菜。传统方法：你让他先学会精确称量每种调料（低 MSE），再教他怎么炒。DFL：你直接让他尝菜，告诉「咸了还是淡了」，让他自己调整调料用量和火候——直接优化最终味道。

**什么时候用**：你控制整个 pipeline（ML + 优化），有足够的数据和算力做端到端训练。

---

## 3. 数据驱动的分布鲁棒优化

> 这其实是本课程 [07 案例](07-case-distributionally-robust.md) 的核心——所以这里只讲和 ML 的交叉。

**核心想法**：你不是假设一个分布再优化——你是用一个 Wasserstein 球包围经验分布，在球内最坏分布下优化。这个框架天然是数据驱动的。

**和 ML 的交叉**：

```
历史数据 ──→ 经验分布 P_hat ──→ Wasserstein 球 B_ε(P_hat) ──→ DRO 模型
                                                                ↓
                                                          out-of-sample 保证
```

**为什么 ML 圈关注**：DRO 提供 **out-of-sample 性能保证**——只要你选的 Wasserstein 半径 ε 足够大，DRO 的解在未见过的数据上也不会差太多。这和 ML 中的泛化理论（generalization bound）有深刻联系。

**生活类比**：你只有 10 个朋友的身高数据（均值 170cm）。你不敢假设「所有人都是正态分布」——你用一个「半径 10cm 的球」包围这 10 个数据点，然后准备一件「在球内最坏情况下也能穿」的衣服。

**什么时候用**：数据少、分布不确定、但需要保证 out-of-sample 表现。

---

## 4. 场景生成 + 深度生成模型

**问题**：传统蒙特卡洛场景生成需要知道分布，矩匹配需要知道矩——如果你连这些都没有呢？

**方案**：用 GAN（生成对抗网络）或 VAE（变分自编码器）从历史数据中学习生成场景。

```
历史需求数据 ──→ GAN 训练 ──→ 生成 10,000 个逼真场景 ──→ 随机规划
```

**优点**：
- 能生成在时间上自相关的场景（比如电力负荷的时间序列）
- 能捕捉非线性关系（比如「需求暴涨」和「运输成本暴涨」同时发生）
- 不需要显式假设分布形状

**缺点**：
- GAN/VAE 训练本身是个不稳定、耗时的过程
- 生成场景的「代表性」可能需要额外验证

**什么时候用**：数据量大但分布复杂、场景之间有复杂的相关性（电力市场、金融时间序列）。

---

## 5. 在线优化 + 带有随机性的 ML

**场景**：你的决策是重复进行的——每天都要决定价格、每周都要补货、每次请求都要分配广告位。而且决策会改变你下一期观察到的数据（这叫做「数据反馈循环」）。

**典型问题**：
- **Contextual Bandit**：依据用户特征（context）选择最优行动，在探索和利用之间平衡
- **Online Convex Optimization**：每期选择策略，结束后观察损失函数
- **Model Predictive Control + 不确定性**：滚动时域优化，每期加入新的观测

**和本课程的关系**：当你重复做随机规划（比如每周跑一次），你是在做在线优化。每周的「新数据」可以用来更新分布假设、调整场景集、或收紧不确定性集合。

> 🆘 逃生通道：这五个方向每个都可以单独写一门课。如果你刚接触这个领域，建议先掌握 **Predict-then-Optimize**（最成熟、最容易落地），再逐步探索 DFL 和 Wasserstein DRO。
