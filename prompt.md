### 01-stochastic-bg.md — 第一部分：不确定性下的优化

**包含：1.1 ~ 1.5 全部内容，约 5,000 字**

#### 1.1 为什么确定性优化不够
- 开场：你把 LP 模型做得完美，结果需求一波动，方案全废了
- **需求不确定**：双十一备货，预测 100 万单，实际 150 万——仓库爆仓、配送瘫痪
- **价格波动**：石油价格一夜涨 30%，运输成本模型全得重算
- **供应链风险**：某港口因台风关闭，你的全球供应链在一周内断裂
- 确定性优化的致命假设：「所有参数已知且不变」——这在真实世界几乎从不成立
- 三个灵魂拷问：我们真的知道未来会怎样？如果不知道，怎么决策？如果知道一些但不是全部，又怎么决策？

> 🆘 核心记住一句话：确定性模型是假设未来是照片，不确定性模型承认未来是视频。

#### 1.2 一段历史：从 Dantzig 到分布鲁棒
- **1955 Dantzig**：提出随机规划——把未来不确定性的概率分布纳入线性规划。当时算力不够，理论超前了几十年
- **1973 Soyster**：提出鲁棒优化——不知道概率分布但知道参数落在某个集合内，求最坏情况下的最优解
- **1990s**：Ben-Tal & Nemirovski 系统化鲁棒优化理论，引入椭球不确定性集合
- **2009 Ben-Tal**：分布鲁棒优化（DRO）的形式化——只知道部分统计信息（均值、方差）
- ASCII 时间轴：

```
1955 ────────── 1973 ──────────── 1990s ────────── 2009 ────→ 现在
Dantzig         Soyster          Ben-Tal &         DRO       ML+
随机规划         鲁棒优化          Nemirovski       形式化     End-to-End
                                 系统化鲁棒                   不确定性优化
```

#### 1.3 不确定性的三种处理方式
一张表讲清楚：

| 方法 | 知道什么 | 不知道什么 | 决策逻辑 | 典型代表 |
|------|---------|-----------|---------|---------|
| 随机规划 | 完整概率分布 | 具体哪个场景会实现 | 在期望意义下最优 | 期望值模型、CVaR |
| 鲁棒优化 | 不确定性的取值范围（集合） | 分布信息 | 在最坏情况下可行 | 箱式/椭球/预算不确定性 |
| 分布鲁棒 | 部分统计信息（均值、方差、支撑集） | 精确分布 | 在最坏可能分布下最优 | Wasserstein DRO、矩模糊 |

**生活类比**：
- **随机规划**：你知道明天有 40% 晴、30% 雨、30% 雪——做决策时要把三种情况都算进去，按概率加权
- **鲁棒优化**：不知道天气概率，但知道温度肯定在 5°C~35°C 之间——你带一件能应付这个区间的外套
- **分布鲁棒**：只知道往年同一天平均温度 20°C、标准差 8°C，但不精确知道分布形状——准备一件 20°C ± 两倍标准差的衣服

#### 1.4 随机规划 vs 鲁棒优化的核心权衡

**保守度 vs 数据需求**

```
                   数据需求高
                       ↑
              随机规划   │
              (需要分布) │
                        │
              ──────────┼──────────→ 保守度
                        │
             分布鲁棒    │
              (需要矩)   │
                       ↓
             鲁棒优化   │
            (需要集合)  │
                        │
                    数据需求低
```

- **随机规划**：需要精确概率分布 → 若分布错了，结果可能比不用还差。典型的生产计划、库存管理
- **鲁棒优化**：只需要不确定性范围 → 但结果可能过于保守。典型的工程结构设计、网络扩容
- **怎么选**：数据质量好、愿意赌分布估计 → 随机规划。数据差、保守一点也能接受 → 鲁棒优化。中间地带 → 分布鲁棒

#### 1.5 建模心法：面对不确定性，怎么开始写模型

五步法：
1. **识别哪些参数不确定**（最重要，2分钟）：打印你的确定性模型，圈出所有「不是完全确定」的系数——需求、价格、成本、可用资源
2. **问自己两个问题**：①你知道参数的概率分布吗？②你能接受多保守的解？
3. **选处理范式**：分布已知 → 随机规划 / 范围已知 → 鲁棒优化 / 部分统计已知 → 分布鲁棒
4. **建模方式**：
   - 随机规划：定义场景 → 写两阶段结构 → 建立对等模型（增大 N 倍的规模）
   - 鲁棒优化：定义不确定性集合 → 推导对等模型（约束数量不变，结构变复杂）
5. **验证保守度**：把结果和确定性模型对比——如果保守到不切实际，调整不确定性集合或放松机会约束

> 🆘 五步法只需要记住前三步。如果连参数不确定都不知道——说明你还没熟悉业务，回去问问运营团队哪些数字经常变。

### 02-methods.md — 第二部分：核心方法

**包含：2.1 ~ 2.6 全部内容，约 6,000 字**

#### 2.1 两阶段随机规划
- **核心结构**：「这里决策 → 观察 → 那里决策」
- 生活类比：双十一备货。第一阶段：你提前两个月定原材料（不知道具体销量）。第二阶段：等预售数据出来，再决定每种产品具体生产多少
- 数学形式：min cᵀx + E[Q(x, ξ)]，其中 Q(x, ξ) = min{qᵀy | Wy ≥ h - Tx, y ≥ 0}
- **非预期约束**（Non-anticipativity）：第一阶段决策不能「看到未来」——你不能在知道需求是多少之后再决定备多少料
- 这里/那里的命名：在随机规划领域，通常把第一阶段叫「这里以及现在」（here-and-now），第二阶段叫「等一下再看」（wait-and-see）

#### 2.2 场景生成
- 蒙特卡洛（Monte Carlo）模拟：从已知分布中随机抽样生成场景。简单，但需要大量样本才能收敛
- **SAA**（Sample Average Approximation）：用抽样平均代替真实期望。核心定理：随着样本量增加，SAA 的解以指数速度收敛到真实最优解
- **矩匹配**（Moment Matching）：生成一组场景使其统计矩（均值、方差、偏度等）匹配目标分布。适合已知均值方差但不知道分布形状的场景
- **情景树生成**：多层决策树，每个节点分叉到若干子节点。树的规模随深度指数增长——深度超过 3 层通常就难以计算了
- **重要性抽样**：对极端场景施加更多权重——与其均匀抽 10000 个场景，不如重点抽 100 个「最危险」的场景

#### 2.3 对等模型（Deterministic Equivalent）
- 生活类比：「把 '万一明天下雨' 的不确定性，转化为 '我带一把伞' 这个确定的决定」
- 核心思路：用一组场景 S = {1, 2, ..., N} 和概率 p_s 代替连续随机变量，把随机问题变成一个巨大的确定性 LP
- 规模膨胀：如果原问题有 n 个变量、m 个约束，N 个场景的对等模型有 ~n + N×n' 个变量、~m + N×m' 个约束
- 计算瓶颈：N 可能上百万——这就是为什么随机规划在 1955 年提出但在 2000 年前后才真正落地
- 分解思想：当 N 太大时，不直接解对等模型，而是用 Benders 分解或 L-shaped 方法

#### 2.4 鲁棒优化的不确定性集合
- 三种核心集合（从简单到复杂）：

| 集合类型 | 形状 | 数学形式 | 保守度 | 计算代价 | 生活类比 |
|---------|------|---------|-------|---------|---------|
| 箱式（Box） | 超矩形 | \|ŝ - ŝ₀\|∞ ≤ ρ | 最高（最保守） | 最低 | 只知道在 0~100 之间 |
| 椭球式（Ellipsoidal） | 椭球 | (ŝ - ŝ₀)ᵀΣ⁻¹(ŝ - ŝ₀) ≤ ρ² | 中等 | 中等 | 知道大概在中心附近，越远可能性越低 |
| 预算式（Budget） | 多面体 | Σᵢ\|ŝᵢ - \hat{s}ᵢ\|/ŝ̂ᵢ ≤ Γ | 可控 | 低 | 最多 K 个参数同时偏离 |

- **预算不确定性**（Budget of Uncertainty）：Bertsimas & Sim (2004) 的关键洞见——并非所有不确定性参数会同时达到最坏值。设预算 Γ = 5，意味最多 5 个需求参数可以同时取边界值。Γ 越大越保守，Γ = 0 退化为确定性模型
- 一句话选择指南：预算不确定性因为线性结构的保持而成为工业界最常用的鲁棒建模方式

#### 2.5 线性优化中的鲁棒对等模型
- 针对每种不确定性集合，给出原始约束和对应的对等约束
- **箱式**：原始约束 Σ aⱼxⱼ ≤ b, aⱼ ∈ [āⱼ - âⱼ, āⱼ + âⱼ] → 对等：Σ āⱼxⱼ + Σ âⱼ|xⱼ| ≤ b
- **预算式**：原始约束同上 → 对等：Σ āⱼxⱼ + max_{S∪{t}} Σ_{j∈S} âⱼ|xⱼ| + (Γ - ⌊Γ⌋)âₜ|xₜ| ≤ b，等价于一组线性约束
- 关键洞见：鲁棒对等模型和原问题**同一类型**（LP → LP，SOCP → SOCP），不会改变问题类别——这是鲁棒优化的工程魅力

#### 2.6 机会约束的近似
- 机会约束定义：P(g(x, ξ) ≤ 0) ≥ 1 - α。即：约束必须被满足的概率不低于 1 - α
- 生活类比：「90% 的情况下不能缺货」——你允许 10% 的情况下缺货
- 机会约束的困难：即使 ξ 是简单的分布，可行集通常非凸
- **三种近似方法**：
  1. **正态假设 + 解析近似**：如果 ξ 服从正态分布，机会约束可以写成均值 + 标准差的倍数 ≤ 0 的形式
  2. **场景近似**：用 N 个场景替换机会约束，要求约束在所有场景中成立。等价于用样本近似分布
  3. **CVaR 近似**：用 CVaR 约束代替机会约束——CVaR 约束是凸的、可解，且比机会约束更保守

> 🆘 记住两点：① 机会约束 = 允许一定概率违反 ② 把机会约束转换成 CVaR 约束是工业界最实用的做法。

## 案例 1：新闻摊贩的随机版 ★★☆☆☆

**方法类型**：随机规划 / 单阶段
**核心概念**：期望值模型、解析解 vs 仿真
**代码**：`code/case01_newsvendor.py`

**场景**：你经营一个报刊亭，每天要决定进货多少份报纸。进货价 c = 0.5 元/份，零售价 p = 1.0 元/份，卖不完的退回价 v = 0.1 元/份。需求 D 是不确定的，服从某个已知分布（均值为 100，标准差为 20 的正态分布）。

**建模**：
- 决策变量：进货量 q（整数）
- 目标：最大化期望利润 E[min(D, q) × p + max(q - D, 0) × v - q × c]
- 关键公式：最优进货量 q* 满足临界分位数 F(q*) = (p - c) / (p - v)，其中 F 是需求的累积分布函数

**教学点**：
- 随机规划中最简单、最有解析解的模型——不是每个随机问题都有解析解
- 解析解 vs 仿真对比：用公式算 q* 和用蒙特卡洛仿真寻优的差异
- 过度订货 vs 订货不足的权衡（underage cost vs overage cost）
- 灵敏度分析：价格变化 10%，最优订货量怎么变？

**✅ 验证标准**：
1. 解析公式算出的 q* 和仿真寻优结果一致（误差 < 1%）
2. 期望利润曲线为凹函数，极值点存在且唯一
3. 需求标准差从 20 增大到 50，最优利润下降且最优订货量变化

**延伸**：
- 多产品新闻摊贩——有预算上限时怎么分配
- 风险规避版本：用 CVaR 代替期望值

## 案例 3：鲁棒投资组合 ★★★☆☆

**方法类型**：鲁棒优化
**核心概念**：不确定性集合、预算不确定性、尾部风险保护
**代码**：`code/case03_robust_portfolio.py`

**场景**：你有 100 万资金要分配到 N=10 只股票上。每只股票的期望收益率 r_i 已知，但实际收益率 r̃_i 会在 [r_i - σ_i, r_i + σ_i] 之间波动（箱式不确定性）。目标：最小化组合在最坏情况下的损失。

**建模**：
- 决策变量：每只股票的投资比例 w_i
- 目标：最大化最坏情况下的组合收益 = max min_{r̃ ∈ U} Σ r̃_i w_i
- 约束：Σ w_i = 1, w_i ≥ 0
- 预算不确定性：最多 Γ 只股票的收益率可以同时偏离到最差值

**教学点**：
- 鲁棒优化的核心：「最坏情况下的最优」
- 三种不确定性集合（箱式/椭球式/预算式）的对比：保守度依次降低
- 预算参数 Γ 的调节：从 Γ = 0（确定性均值模型）到 Γ = N（最保守的箱式模型）
- 对比均值-方差模型（Markowitz）vs 鲁棒模型的区别

**✅ 验证标准**：
1. Γ = 0 时鲁棒模型 = 等权组合
2. Γ 增大 → 组合收益变保守（最坏情况收益增大，期望收益下降）
3. 回溯测试：鲁棒组合在最坏月份比均值-方差组合亏得更少

**延伸**：
- 椭球不确定性集合下的 SOCP 模型
- 引入 CVaR 约束的事后对比

## 案例 5：分布鲁棒优化 ★★★★☆ [进阶]

**方法类型**：分布鲁棒优化
**核心概念**：Wasserstein 模糊集合、矩模糊、最坏情况分布
**代码**：`code/case05_distributionally_robust.py`

**场景**：回到新闻摊贩问题。你不知道需求的精确分布，只知道历史上 100 天的需求数据。均值 μ = 100，方差 σ² = 400，但分布不一定是正态的——可能有偏态、厚尾。你希望在「所有与数据在 Wasserstein 距离 ≤ ε 的分布中，最坏情况下的期望利润最大」。

**建模**：
- Wasserstein 模糊球：B_ε(P_hat) = {Q: d_W(Q, P_hat) ≤ ε}
- P_hat 是经验分布（100 个样本点的等权重分布）
- DRO 模型：max min E_Q[profit(q, D)] over Q ∈ B_ε(P_hat)
- 目标：在模糊球内最坏分布下的最优订货量

**教学点**：
- 为什么分布鲁棒比随机规划更稳健、比鲁棒优化更灵活
- Wasserstein 距离的直觉：两个分布之间需要「搬多少概率质量」
- 半径 ε 的调节：ε = 0 → 退化为随机规划（精确分布），ε → ∞ → 退化为鲁棒优化
- 数据驱动：用 100 个数据点直接构建模糊球，不用假设分布

**✅ 验证标准**：
1. ε = 0 时 DRO 解 ≈ 随机规划解（用经验分布）
2. ε 增大 → 订货量趋于保守（更接近鲁棒解）
3. 在真实分布非正态时，DRO 解比随机规划解有更好的 out-of-sample 表现

**延伸**：
- 矩模糊（Mean-Covariance ambiguity set）——只知道均值和协方差矩阵
- DRO 在金融风控中的应用

| 场景数巨大（> 10⁶）且无法分解 | 仿真优化 / 元启发式 | 对等模型规模太大，直接求解不可行 |
| 决策是即时响应（毫秒级） | 基于规则的策略 | 优化模型跑完需求已经变了 |
| 只需要一个「够好」的解 | 鲁棒优化（用预算不确定性） | 比随机规划简单，比确定性安全 |

---

### appendix-b-common-pitfalls.md — 附录B：TOP 5 必踩坑

1. **场景太少 → 低估风险**：5 个场景没法代表真实分布——最少 50 个场景起步（除非有解析解）
2. **分布假设错误 → 方向性偏误**：假设正态而实际厚尾——尾部事件会频繁打脸。先做分布检验
3. **鲁棒优化过度保守 → 方案不可用**：箱式 + 所有参数同时最差 → 方案保守到没人用。用预算不确定性控制保守度
4. **忘记非预期约束**：第一阶段决策不能知道第二阶段的信息——这不是数学问题，是逻辑问题。不写非预期约束，模型会「作弊」
5. **忽略计算代价 → 模型跑不出结果**：10 个场景 × 100 个变量 ≈ 1000 变量没问题，10000 个场景 → 问题规模失控。先用小规模验证逻辑

---

### appendix-c-ml-intersection.md — 附录C：随机规划 × ML 的交叉点

1. **Predict-then-Optimize**：ML 预测需求分布 → 随机规划优化决策。端到端框架（Elmachtoub & Grigas, 2021 的 Smart Predict-then-Optimize）
2. **Decision-Focused Learning**：训练 ML 模型直接最小化下游决策损失（SPO loss, DFL）
3. **数据驱动的分布鲁棒优化**：用 Wasserstein DRO 连接经验数据和优化决策——保证 out-of-sample 表现
4. **情景生成 + 深度生成模型**：GANs / VAE 用于生成更真实的业务场景（供应链、电力市场）
5. **在线优化 + 带随机性的 ML**：在线凸优化、上下文随机 bandit 问题中的不确定性处理
