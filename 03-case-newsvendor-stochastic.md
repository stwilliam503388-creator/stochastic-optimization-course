# 案例 1：报童问题的随机规划解法

> 难度 ★★☆☆☆ · 随机规划 / 单阶段 · 核心概念：期望值模型、解析解 vs 仿真 · 代码：`code/case01_newsvendor.py`

---

## 1 场景

你经营一个报刊亭，每天早上 6 点得决定今天进多少份报纸。

进货价 **c = 0.5 元/份**，零售价 **p = 1.0 元/份**，卖不完的退回给报社还能拿回 **v = 0.1 元/份**。今天到底有多少人买报纸——你猜不准。

根据历史数据，需求 D 服从均值为 100、标准差为 20 的正态分布（整数）。订多了，剩下一堆废纸；订少了，眼睁睁看着顾客空手走。

**问题**：订多少份，能让我平均下来赚得最多？

---

## 2 建模

### 变量

- **决策变量**：进货量 q（整数，≥ 0）
- **随机变量**：需求 D，D ~ N(100, 20²)

### 目标

最大化**期望利润**：

```
利润(q, D) = 销售收入 + 退货收入 - 进货成本

利润(q, D) = p × min(q, D) + v × max(q - D, 0) - c × q
```

因为 D 随机，利润也是随机的。我们对 D 取期望：

```
max E_D[ p × min(q, D) + v × max(q - D, 0) - c × q ]
```

### 解析最优解

这个问题的妙处在于**它有解析解**——不是每个随机问题都这么友好。

最优订货量 q\* 满足：

```
F(q*) = (p - c) / (p - v)
```

其中 F(·) 是需求的累积分布函数（CDF）。

代入数字：**临界分位数 = (1.0 - 0.5) / (1.0 - 0.1) ≈ 0.556**

查正态分布表：q\* ≈ 100 + 0.14 × 20 ≈ **103 份**。

> **直觉**：临界比 (p-c)/(p-v) 衡量的是「少订一份损失的利润」vs「多订一份浪费的成本」之间的平衡。0.556 说明少订的风险略大于多订的风险，所以最优订货量略高于均值。

---

## 3 方法详解

### 方法 A：解析法（公式直接算）

用临界分位数公式加正态分布表，30 秒出结果。本质上是**单阶段随机规划**的闭式解。

### 方法 B：样本均值近似（SAA）

不知道分布或者求不出解析解时，用蒙特卡洛模拟：

1. 从 N(100, 20²) 中抽取 N 个需求场景 D₁, D₂, ..., D_N
2. 用样本均值替代期望值：

```
max (1/N) × Σ [p × min(q, D_s) + v × max(q - D_s, 0) - c × q]
```

3. 对一组候选 q 值暴力搜索，选利润最高的那个

### 方法 C：灵敏度分析

把标准差从 20 改成 50，看看最优订货量和利润怎么变——用来理解「不确定性越大，生意越难做」这个直觉。

## 4 代码实现概览

```python
# 关键代码（完整版见 code/case01_newsvendor.py）

import numpy as np
from scipy.stats import norm

c, p, v = 0.5, 1.0, 0.1          # 成本参数
mu, sigma = 100, 20              # 需求分布
critical_ratio = (p - c) / (p - v)
q_star = int(norm.ppf(critical_ratio, mu, sigma))

def expected_profit(q, mu, sigma, n_scenarios=100000):
    """通过蒙特卡洛模拟计算期望利润"""
    D = np.random.normal(mu, sigma, n_scenarios)
    sales = np.minimum(q, D)
    returns = np.maximum(q - D, 0)
    profit = p * sales + v * returns - c * q
    return profit.mean()

def deterministic_solution():
    """对比：用均值需求作为已知数的确定性方案"""
    q_det = int(mu)  # 确定性方案：进均值那么多
    return q_det, expected_profit(q_det, mu, sigma)
```

## 6 关键洞察

1. **解析解是幸运，不是常态**。报童问题是少数有闭式解的随机优化问题——大部分实际问题的期望值函数没法写成简单公式，得靠 SAA + 搜索或者规划求解器。

2. **少订 vs 多订的成本不对称**。少订一份损失 (p-c) = 0.5 元，多订一份浪费 (c-v) = 0.4 元。少订更贵→最优 q > 均值。这就是临界比的本质——它是两个成本的比值。

3. **确定性方案永远不最优**。用均值需求做决策听起来合理，但期望利润一定低于随机规划解。这叫**随机规划的价值（VSS）**。

4. **不确定性越大，利润越低，但备货反而要更多**。直觉上波动大应该保守——不对，因为少订的代价更大时，你得增加库存来兜底。

## 7 延伸与变形

1. **多产品报童**——有预算上限时，怎么在多种报纸之间分配采购量？变成一个随机背包问题。

2. **风险规避版本**——用 CVaR 代替期望值。你会倾向于订更少的货（减少尾部损失），但代价是期望利润下降。

3. **连续补货**——如果不止订一次，而是可以动态补货，就变成 (s, S) 库存策略问题。

4. **分布鲁棒版**——如果不知道具体分布只知道均值和方差，怎么订？→ 见案例 5。
