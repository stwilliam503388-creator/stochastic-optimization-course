## 1 场景

回到你熟悉的报童问题框架，但这次我们用**四种不同的思维方式**来解决同一个问题，然后对比它们在真实世界中的表现。

### 问题设定

你是一家小型制造企业的运营总监，需要为下个季度的生产**提前备料**。

- 产品预售价：**p = 100 元/件**
- 原材料成本：**c = 40 元/件**
- 未售出残值：**v = 10 元/件**
- 缺货惩罚（品牌损失）：**g = 20 元/件**

需求不确定。根据你手头的信息，需求均值为 μ = **200**，标准差为 σ = **50**。

但问题是——**你对分布形状没把握**。它可能是正态、均匀、厚尾 t 分布、或者偏态分布。

你会用哪套方法来做决策？

## 2 四种方法建模

### 方法 1：确定性方案（Det）

**思路**：假装需求就是均值 200，按这个数备货。

```
模型：max  p·min(q, 200) + v·max(q-200, 0) - c·q - g·max(200-q, 0)
```

```
最优 q_det = 200（就是均值）
期望利润 = p·200 - c·200 = 12,000 元
```

问题：这是「天真」的期望——实际需求不会是 200，利润会打折。

### 方法 2：随机规划（SP）

**思路**：假设需求服从正态分布 N(200, 50²)，用 SAA 求解。

```
最优 q*_sp = F^{-1}( (p + g - c) / (p + g - v) )
          = F^{-1}( (100+20-40) / (100+20-10) )
          = F^{-1}(0.7273) ≈ 230 件
```

随机规划说：多备 30 件！因为缺货的代价（80 元/件）远大于积压的代价（30 元/件）。

### 方法 3：鲁棒优化（RO）

**思路**：不知道分布，只知道需求在 [50, 350] 之间。在最坏情况下做最优决策。

```
箱式鲁棒：max_q  min_{D∈[50,350]} 利润(q, D)

最坏情况：需求 = 50（卖不出去！）
所以 RO 不建议你备太多货...
```

```
最优 q*_ro ≈ 115 件
```

鲁棒优化说：只备 115 件。因为最坏情况是需求惨淡，你不想压一堆货在手上。

### 方法 4：分布鲁棒优化（DRO）

**思路**：只知道 μ=200, σ=50，用 Scarf 公式。

```
q*_dro = μ + (σ/2) × ( √((p+g-v)/(p+g-c)) - √((p+g-c)/(p+g-v)) )
       = 200 + 25 × ( √(110/80) - √(80/110) )
       = 200 + 25 × (1.1726 - 0.8528)
       = 200 + 25 × 0.3198
       = 208 件
```

DRO 说：备 208 件——比 SP 少 22 件，比 RO 多 93 件。

### 四种方法对比总表

| 方法 | 需要的信息 | 订货量 | 保守度 | 逻辑 |
|------|-----------|-------|-------|------|
| 确定性 | 均值 | 200 | ⭐无防护 | 当没不确定性 |
| 随机规划 | 完整分布 | 230 | ⭐低 | 赌分布是对的 |
| 分布鲁棒 | 均值+方差 | 208 | ⭐⭐中 | 防分布假设错误 |
| 鲁棒优化 | [min, max] | 115 | ⭐⭐⭐高 | 防最坏情况 |

## 3 方法详解

### 验证框架

要想比较四种方法，不能只看它们各自假设下的结果——那是在「自己证明自己」。

正确做法：

1. 设定一个**不知道的「真实分布」**（比如实际上需求是 t 分布）
2. 用四种方法算出各自的订货量 q_det、q_sp、q_dro、q_ro
3. 在真实分布下跑蒙特卡洛模拟，计算每种策略的实际期望利润、最差 5% 分位利润、以及最坏情况利润
4. 换一个真实分布，重复

### 测试的真实分布

| 真实分布 | 形状 | 特征 |
|---------|------|------|
| 正态 | 对称钟形 | SP 的假设正确 |
| 均匀 | 平顶 | 没有极端值 |
| t(3) 厚尾 | 尖峰+厚尾 | 极端值频繁出现 |
| 偏态 (对数正态) | 右偏 | 均值≠中位数 |

## 4 代码实现概览

```python
# 关键代码（完整版见 code/capstone.py）

import numpy as np
from scipy.stats import norm, t, lognorm, uniform

# === 四种策略 ===
def solve_deterministic(c, p, v, g, mu):
    return mu  # 确定性：按均值备货

def solve_stochastic(c, p, v, g, mu, sigma):
    cr = (p + g - c) / (p + g - v)
    return int(norm.ppf(cr, mu, sigma))

def solve_robust(c, p, v, g, d_min, d_max):
    # 箱式鲁棒的近似求解...
    return int((p + g - c) / (p + g - v) * d_min + ...)

def solve_dro(c, p, v, g, mu, sigma):
    ratio = np.sqrt((p + g - v) / (p + g - c))
    return mu + (sigma / 2) * (ratio - 1/ratio)

# === 验证 ===
def evaluate(q, dist_type, n_sim=100000):
    """在真实分布下计算策略表现"""
    if dist_type == 'normal':
        D = np.random.normal(200, 50, n_sim)
    elif dist_type == 't3':
        D = np.random.standard_t(3, n_sim) * 50/np.sqrt(3) + 200
    elif dist_type == 'uniform':
        D = np.random.uniform(60, 340, n_sim)
    elif dist_type == 'lognormal':
        D = np.random.lognormal(np.log(200) - 0.5*0.25**2, 0.25, n_sim)
    # 计算利润
    profit = p * np.minimum(q, D) + v * np.maximum(q - D, 0) \
           - c * q - g * np.maximum(D - q, 0)
    return profit.mean(), np.percentile(profit, 5), profit.min()
```

## 6 关键洞察

1. **没有银弹**。没有哪个方法在所有场景下都是最优的。选方法取决于：
   - 你对分布有多确定（精确分布 → SP，部分信息 → DRO，完全不知道 → RO）
   - 你对尾部风险的容忍度（不能接受暴亏 → RO，可以接受平均最优 → SP）

2. **DRO 是「实战中最容易推荐」的方法**。从运行结果看，DRO 在四种分布下都没拿过倒数第一，拿了两次 🥇。它不是最激进的，但几乎总是最稳健的——在真实商业场景中，「稳健」比「最优」更值钱。

3. **确定性方案才是你真正的对手**。大部分公司今天的决策就是确定性方案（用均值预测）。证明 SP/DRO/RO 能比它好，就是你的商业价值。本案例中确定性方案在所有场景下都是 🥉或 4——这就是**随机规划的价值（VSS）**。

4. **保守度不是「好」或「坏」，而是工具**。RO 在正态分布下差了 12%，但如果你是一个不愿意承受任何重大损失的高管，RO 的「最坏情况保护」对你来说值得花这个溢价。

## 7 延伸与变形

1. **混合策略**——不用一个方法一票到底，而是在不同场景下自适应切换方法。比如，先用 DRO 决策，在观察到部分信息后转为 SP。

2. **贝叶斯更新**——随着销售季进行，需求数据不断进来。用贝叶斯方法更新分布信息，逐步从 DRO 过渡到 SP。

3. **多周期 + 多产品**——把毕业项目扩展到 3 个周期、20 种产品。你会看到 SP 的规模爆炸问题，以及 RO/DRO 在计算上的优势。

4. **面对老板的汇报话术**：
   - 用确定性方案做基准线（"我们现在就是这样做的"）
   - 用随机规划展示「如果信息准确能多赚多少」
   - 用鲁棒优化展示「我们最差能承受多少损失」
   - 用分布鲁棒展示「落在期望区间外的概率」
