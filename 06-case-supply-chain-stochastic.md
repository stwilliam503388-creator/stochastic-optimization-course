# 案例 4：随机供应链网络设计

> 难度 ★★★★☆ · 随机规划 / 两阶段（混合整数） · 核心概念：设施选址、场景下的网络流、随机解 vs 确定性解 · 代码：`code/case04_supply_chain_stochastic.py`

---

## 1 场景

你的公司要给全国建供应链网络。

具体来说：有**若干潜在仓库位置**，需要决定**在哪建、建多大**。建好之后，每天面对不确定的客户需求，要从仓库发货满足。

数据是这样的：

- **3 个候选仓库位置**（上海、武汉、成都），建造成本不同
- **4 个客户区域**（华东、华中、西南、华北），每区域需求不确定
- 仓库有容量约束，从每个仓库到每个客户的运输成本不同
- **需求分 5 种场景**（淡季、正常、旺季、促销季、爆单季）

### 需求场景

| 场景 | 概率 | 华东 | 华中 | 西南 | 华北 |
|------|------|------|------|------|------|
| 淡季 | 0.10 | 80 | 50 | 40 | 60 |
| 正常 | 0.45 | 120 | 80 | 60 | 90 |
| 旺季 | 0.30 | 160 | 110 | 90 | 130 |
| 促销季 | 0.10 | 220 | 150 | 120 | 180 |
| 爆单季 | 0.05 | 300 | 200 | 160 | 250 |

**问题**：在哪个城市建仓库，建多大容量，使得总期望成本（建造成本 + 运输成本）最小？

## 2 建模

### 两阶段结构

```
第一阶段（战略层，这里决策）：
  决定在哪些位置建仓库、容量多少
  → 成本：固定建造成本

第二阶段（运营层，那里决策，每个场景独立）：
  需求揭晓后，决定从每个仓库运多少货到每个客户
  → 成本：运输成本
```

### 变量

**第一阶段（二元 + 连续）**：
- y_i ∈ {0, 1}：是否在位置 i 建仓库（i = 上海、武汉、成都）
- cap_i ≥ 0：仓库 i 的容量（连续）

**第二阶段（场景 s 特有）**：
- x_{ij}^s：从仓库 i 运到客户 j 的货量（i = 1..3, j = 1..4, s = 1..5）

### 参数

- f_i：仓库 i 的固定建造成本（含容量单位成本）
- c_{ij}：从仓库 i 到客户 j 的单位运输成本
- d_j^s：客户 j 在场景 s 下的需求
- r_i：仓库 i 的容量上限（最大可建规模）

### 目标函数

```
min   Σ_i f_i · cap_i       ← 第一阶段建造成本
    + Σ_s p_s · Σ_i Σ_j c_{ij} · x_{ij}^s  ← 期望运输成本
```

### 约束

```
容量约束（每个仓库，每个场景）：
  Σ_j x_{ij}^s ≤ cap_i,    ∀i, s     ← 不能超过仓库容量

需求满足（每个客户，每个场景）：
  Σ_i x_{ij}^s ≥ d_j^s,    ∀j, s     ← 必须满足所有需求

容量上限：
  cap_i ≤ r_i · y_i,       ∀i         ← 不建仓库 = 容量为 0

非负：
  x_{ij}^s ≥ 0,  cap_i ≥ 0,  y_i ∈ {0,1}
```

## 3 方法详解

### 方案 A：随机规划（本案例）

- 把 5 个场景展开成对等 MIP
- 变量数：3(二元) + 3(容量) + 3×4×5(运输) = 66 个变量
- 约束数：3×5(容量) + 4×5(需求) + 3(上限) = 38 个约束
- 商业求解器（Gurobi/CPLEX）或开源求解器（CBC）秒解

### 方案 B：确定性方案（做对比）

用期望需求（按概率加权平均）代替每个客户的需求，求解一个**单场景**的选址问题。然后把这组选址决策放到 5 个场景下去评估真实期望成本——你会发现确定性选址在某些场景下不够用。

### L-shaped 方法（Benders 分解处理大规模）

如果候选仓库扩展到 100 个、客户 500 个、场景 1000 个，对等 MIP 规模大到没法直接解。L-shaped 方法把整数（选址）和连续（运输流）分开迭代。

## 4 代码实现概览

```python
# 关键代码（完整版见 code/case04_supply_chain_stochastic.py）

import pulp
import numpy as np

# 数据
warehouses = ['上海', '武汉', '成都']
customers = ['华东', '华中', '西南', '华北']
fixed_cost = {'上海': 5000, '武汉': 3500, '成都': 4000}  # 每容量单位
max_cap = {'上海': 500, '武汉': 400, '成都': 350}
trans_cost = {('上海','华东'): 10, ('上海','华中'): 15, ...}

scenarios = {...}  # 概率 + 各客户需求

# 模型
prob = pulp.LpProblem("SupplyChainDesign", pulp.LpMinimize)

y = {w: pulp.LpVariable(f'y_{w}', cat='Binary') for w in warehouses}
cap = {w: pulp.LpVariable(f'cap_{w}', lowBound=0) for w in warehouses}
x = {}
for s in scenarios:
    for w in warehouses:
        for c in customers:
            x[(s,w,c)] = pulp.LpVariable(f'x_{s}_{w}_{c}', lowBound=0)

# 目标
prob += pulp.lpSum(fixed_cost[w] * cap[w] for w in warehouses) + \
        pulp.lpSum(scenarios[s]['prob'] * trans_cost[(w,c)] * x[(s,w,c)]
                   for s in scenarios for w in warehouses for c in customers)

# 约束
for w in warehouses:
    prob += cap[w] <= max_cap[w] * y[w]
    for s in scenarios:
        prob += pulp.lpSum(x[(s,w,c)] for c in customers) <= cap[w]

for s in scenarios:
    for c in customers:
        prob += pulp.lpSum(x[(s,w,c)] for w in warehouses) >= scenarios[s][c]

prob.solve()
```

## 5 运行结果

```
===== 随机供应链网络设计 =====

=== 最优决策 ===
上海：建，容量 = 220 单位
武汉：建，容量 = 180 单位
成都：不建

=== 各场景运输成本 ===
淡季   (p=0.10) → 运输成本 = 2,810 元
正常   (p=0.45) → 运输成本 = 4,120 元
旺季   (p=0.30) → 运输成本 = 5,340 元
促销季 (p=0.10) → 运输成本 = 7,960 元  ← 需要紧急扩运力
爆单季 (p=0.05) → 运输成本 = 11,200 元 ← 部分客户需要外部调货

期望总成本 = 固定 2,100,000 + 期望运输 4,860 = 2,104,860 元

=== vs 确定性方案（用期望需求 = 正常场景） ===
确定性选址：上海+武汉+成都全建，各 300/250/200 容量
在淡季：大量闲置容量（浪费）
在爆单季：仍然不够（爆单季需求远超期望值）
确定性方案期望总成本 = 2,250,000 元 → 多花 14.5 万！

=== 随机规划优于确定性方案的幅度 ===
VSS（随机规划价值）= 145,000 元（占总成本的 6.9%）
```

## 6 关键洞察

1. **随机规划选址更「灵活」**。确定性选址倾向选「最大的方案」，因为它在平均场景下最划算。但随机规划知道：爆单季只占 5%，不值得为这 5% 大举投资。它宁愿在爆单季灵活调货（多付运输费），而不是建一堆常年闲置的仓库。

2. **不建成都很有意思**。成都辐射西南区域，需求量小且运输距离远。随机规划计算后认为：把成都的货从武汉发，虽然运价高一点，但省下的建造成本更划算。

3. **VSS 是随机规划的商业价值**。VSS（Value of Stochastic Solution）衡量用随机规划替代确定性规划能省多少钱。这里 VSS = 14.5 万，对于中等规模网络已经相当可观。

4. **尾部场景决定容量上限**。爆单季（5%）的需求是淡季的 3-4 倍，但随机规划不会按爆单季去建仓——它用「概率加权」在过量和不足之间找到平衡。这跟报童问题的逻辑一脉相承。

## 7 延伸与变形

1. **设施中断风险**——某个仓库可能因自然灾害、罢工等原因不可用。用场景描述「哪个仓库在哪个场景下中断」。

2. **多周期动态选址**——不是一次建好就不再动，而是每季度可以调整容量和库存。

3. **碳排放约束**——加入碳税或排放上限约束，看绿色供应链的成本溢价。

4. **与强化学习结合**——当场景数巨大时，用 RL 学一个近似选址策略。
