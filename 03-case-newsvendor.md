# 案例 1：新闻摊贩的随机版

> 难度 ★★☆☆☆ · 方法：随机规划/单阶段 · 核心概念：期望值模型、解析解 vs 仿真 · 代码：[code/case01_newsvendor.py](code/case01_newsvendor.py)

---

## 1 场景

你经营一个报刊亭。每天要决定进货多少份报纸。数字如下：

| 参数 | 值 |
|------|-----|
| 进货价 c | 0.5 元/份 |
| 零售价 p | 1.0 元/份 |
| 退货价 v | 0.1 元/份 |
| 需求 D ~ N(μ=100, σ=20) | 均值为 100，标准差为 20 的正态分布 |

**关键矛盾**：
- 进货太少 → 错失销售机会（underage cost = p - c = 0.5 元/份）
- 进货太多 → 卖不完折价退（overage cost = c - v = 0.4 元/份）

> 🆘 核心记住：所有随机优化问题都在这两种「错误」之间权衡。

---

## 2 建模：变量 → 约束 → 目标

**决策变量**：进货量 q（整数，≥ 0）

**目标**：最大化期望利润

```
max  E[ profit(q, D) ]
```

其中：
```
profit(q, D) = min(D, q) × p + max(q - D, 0) × v - q × c
```

**逐符号翻译**：
- `min(D, q) × p`：卖出去的部分 × 售价
- `max(q - D, 0) × v`：没卖出去的部分 × 退货价
- `q × c`：进货成本

---

## 3 方法详解

### 3.1 解析解（唯一可以手算的随机规划问题）

**关键公式**：
```
F(q*) = (p - c) / (p - v) = critical ratio
```

其中 F 是需求 D 的累积分布函数（CDF），critical ratio = 临界比率。

**代入数字**：
- critical ratio = (1.0 - 0.5) / (1.0 - 0.1) = 0.5 / 0.9 ≈ 0.5556
- 查正态分布 CDF：F⁻¹(0.5556) ≈ 0.14（标准正态分位数）
- q* = μ + 0.14 × σ = 100 + 0.14 × 20 ≈ 102.8 → 取整 103 份

**含义**：当日需求累积分布达到 55.56% 时的需求量就是最优进货量。

### 3.2 仿真方法

当问题变复杂（多产品、非线性成本），解析解就不存在了。这时候用蒙特卡洛仿真：

1. 生成 N 个需求样本 D₁, ..., D_N ~ N(100, 20)
2. 对每个候选 q，计算平均利润
3. 搜索使期望利润最大的 q

## 6 关键洞察

**为什么新闻摊贩在随机规划中地位那么高？**
- 它是**唯一有解析解的随机规划模型**——让你能直观理解随机优化的核心权衡
- 临界比率公式可以推广到大部分单周期库存问题
- 几乎所有更复杂的模型（多产品、多阶段、风险规避）都是在这个模型上做加法

**过度订货 vs 订货不足的权衡线**：
- critical ratio > 0.5：进货偏多（因为少卖损失 > 多进货损失）
- critical ratio < 0.5：进货偏少（反之）
- critical ratio = 0.5：两种损失一样，进货量 = 需求中位数

---

## 7 延伸与变形

- **多产品新闻摊贩**：有预算上限时怎么分配——这是一个随机背包问题
- **风险规避版本**：用 CVaR 代替期望值，进货量更保守
- **需求预测更新**：先用先验分布，然后根据早期销售数据更新后验（贝叶斯新闻摊贩）
