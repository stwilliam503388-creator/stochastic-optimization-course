# 第一部分：不确定性下的优化

> 术语速查见 `README.md`

## 1.1 为什么确定性优化不够

**开场问题：**

你把一个 LP 模型做得完美——变量定义清晰、约束严谨、目标函数合理。模型跑出来说：「备货 100,000 件，预期利润 500 万。」

然后双十一实际需求 150,000 件。仓库爆仓、配送瘫痪、你紧急加单的物流成本吃掉了所有利润。

**问题出在哪？**

你假设「需求 = 已知常数 100,000」。但你知道的——需求从来不是常数。

### 现实世界的三重不确定性

**① 需求不确定**：双十一备货，预测 100 万单，实际 150 万——整个供应链崩溃。这不是预测不准的问题，这是确定性模型假设「未来是已知的」，但真实世界是「未来是个概率分布」。

**② 价格波动**：你今天排好的运输计划，明天石油涨价 30%，整个成本模型重算。你的 LP 模型里有「运输成本 = 5 元/公里」——但真实世界运输成本在波动。

**③ 供应链风险**：某港口因台风关闭、某个供应商突然断货、某个产线检修延长——任何一环出问题，最优解就不再是最优解。

### 确定性优化的致命假设

```
确定性模型：「所有参数已知且不变」
          ↓
真实世界：  ❌ 需求会变  ❌ 价格会变  ❌ 供应会断
          ↓
模型输出：「备货 100,000 → 利润 500 万」
             ↓
实际结果：  「需求 150,000 → 爆仓 → 实亏 200 万」
```

> 🆘 核心记住一句话：**确定性模型是假设未来是照片，不确定性模型承认未来是视频。**

### 三个灵魂拷问

在你继续往下读之前，问自己三个问题：

1. **我们真的知道未来会怎样？** → 几乎从不。如果你真的知道——那是预测，不是决策。
2. **如果不知道，怎么决策？** → 你需要一个「对不确定性鲁棒」的方案，而不是一个「假设确定」的方案。
3. **如果知道一些但不是全部，又怎么决策？** → 这才是大部分真实场景——你知道一些（需求大概在哪个范围、平均多少），但不精确知道全部。这就是本课程要解决的核心问题。

## 1.2 一段历史：从 Dantzig 到分布鲁棒

> 这不是历史课——但理解这段历史，你就理解了为什么有「三种处理不确定性的方式」。

```
1955 ────────── 1973 ──────────── 1990s ────────── 2009 ────→ 现在
Dantzig         Soyster          Ben-Tal &         DRO       ML+
随机规划         鲁棒优化          Nemirovski       形式化     End-to-End
                                 系统化鲁棒                   不确定性优化
```

**1955 — Dantzig 提出随机规划（Stochastic Programming）**
George Dantzig（线性规划之父）发表了第一篇随机规划论文。核心洞见：把未来不确定性的概率分布直接纳入线性规划。当时的问题是——算力不够。一台 IBM 701 每秒只能做几千次运算，随机规划的对等模型规模是原始问题的 N 倍（N 是场景数）。理论超前了几十年。

**1973 — Soyster 提出鲁棒优化（Robust Optimization）**
A.L. Soyster 提出了另一个思路：如果你不知道概率分布，但知道参数落在某个集合内，能不能保证方案在最坏情况下也是可行的？这就是鲁棒优化的起点。他提出的「箱式不确定性」后来被证明过于保守（因为不可能所有参数同时最差），但方向是对的。

**1990s — Ben-Tal & Nemirovski 系统化鲁棒优化理论**
Aharon Ben-Tal 和 Arkadi Nemirovski 把鲁棒优化从单一概念发展成一整套数学框架。他们引入了椭球不确定性集合——形状像椭球的不确定集，比箱式更紧、没那么保守。他们还证明了鲁棒对等模型和原问题是同一类型（LP → LP，SOCP → SOCP）。

**2009 — 分布鲁棒优化（DRO）的形式化**
DRO 的雏形在 2000 年代初期就出现了，但 Daniel Kuhn 和团队在 2009 年前后将其形式化。核心思想：你不知道精确分布，但知道一些统计信息（均值、方差）——那么怎么在最坏可能分布下做最优决策？

**现在 — ML + End-to-End 不确定性优化**
深度学习和强化学习正在改变不确定性优化的实现方式。Predict-then-Optimize 框架用 ML 预测分布、用优化做决策；Decision-Focused Learning 让 ML 模型直接最小化下游决策损失。附录 C 会有更多。

## 1.3 不确定性的三种处理方式

**先看这张表——这是整门课程的大纲地图。**

| 方法 | 知道什么 | 不知道什么 | 决策逻辑 | 典型代表 |
|------|---------|-----------|---------|---------|
| 随机规划 | 完整概率分布 | 具体哪个场景会实现 | 在期望意义下最优 | 期望值模型、CVaR |
| 鲁棒优化 | 不确定性的取值范围（集合） | 分布信息 | 在最坏情况下可行 | 箱式/椭球/预算不确定性 |
| 分布鲁棒 | 部分统计信息（均值、方差、支撑集） | 精确分布 | 在最坏可能分布下最优 | Wasserstein DRO、矩模糊 |

### 生活类比

想象你明天要出门，但天气预报不准。

**随机规划**：你知道明天有 40% 概率晴、30% 雨、30% 雪。你做决定时要算清每种天气的预期后果——带伞期望收益最大？还是三种情况都穿防水鞋？你把所有可能性按概率加权，做期望最优的决定。

**鲁棒优化**：不知道天气概率，但知道温度肯定在 5°C~35°C 之间。你带一件能应付这个区间的外套——可能不是最完美的选择（不会专门为 25°C 穿最薄的衣服），但你保证在任何情况下都不会冻着或热着。

**分布鲁棒**：你知道往年同一天平均温度 20°C、标准差 8°C，但不精确知道分布形状——也许它在某些年份有极端值。你准备一件「20°C ± 两倍标准差」的衣服——你选了最保守的答案，但并不假设所有往年数据都是等可能的。

### 工作类比

**随机规划**：下季度需求知道三种情景及其概率——高增长 1200（30%）、平稳 1000（50%）、衰退 800（20%）。备货方案按期望利润最优来定。但在真实世界，需求分布很少是精确已知的——这是随机规划最大的弱点。

**鲁棒优化**：你只知道需求在 800 到 1200 之间。你设计一个方案，保证在 800 到 1200 之间的任何一个需求值下都是可行的。方案可能保守（多建了点产能），但永远不会因为需求超预期而出现供应缺口。

**分布鲁棒**：你知道历史需求均值 1000、方差 20000，知道需求非负——但不太确定分布是正态还是偏态。你找一个「对最坏可能分布也是最优」的方案。这在数据量少（新业务）时特别有用。

## 1.4 随机规划 vs 鲁棒优化的核心权衡

**保守度 vs 数据需求**

```
                   数据需求高
                       ↑
              随机规划   │
              (需要分布) │
                        │
              ──────────┼──────────→ 保守度
                        │
             分布鲁棒    │
              (需要矩)   │
                       ↓
             鲁棒优化   │
            (需要集合)  │
                        │
                    数据需求低
```

### 具体解释

**随机规划**：需要精确概率分布 → 如果分布错了，结果可能比不用还差。
- **适用场景**：你有历史数据、做了分布拟合、相对确定分布假设基本正确
- **典型领域**：生产计划、库存管理、电力市场
- **风险**：分布假设错了 → 方案不优甚至更差

**鲁棒优化**：只需要不确定性范围 → 但结果可能过于保守。
- **适用场景**：数据少或分布未知，但能定义合理范围
- **典型领域**：工程结构设计、网络扩容、有安全要求的系统
- **风险**：过于保守 → 方案成本太高、竞争不过

**怎么选？**

| 你的情况 | 推荐方法 |
|----------|---------|
| 数据质量好、有精确分布 | 随机规划 |
| 对分布没把握但能定义范围 | 鲁棒优化 |
| 中间地带——有部分统计信息 | 分布鲁棒 |
| 只要一个「够好」的方案 | 鲁棒优化（预算不确定性） |
| 需要方案在极端情况下仍然安全 | 鲁棒优化 |

## 1.5 建模心法：面对不确定性，怎么开始写模型

> 🆘 五步法只需要记住前三步。如果连「参数不确定」都不知道——说明你还没熟悉业务，回去问问运营团队哪些数字经常变。

### Step 1：识别哪些参数不确定（最重要——花 2 分钟）

把你现有的确定性模型打印出来，圈出所有「不是 100% 确定的」系数：

| 参数类型 | 常见不确定参数 | 确定性假设 |
|---------|--------------|-----------|
| 需求 | 市场需求、客户订单 | 需求 = 常数 |
| 价格 | 原材料价格、产品售价 | 价格 = 常数 |
| 成本 | 运输成本、制造成本 | 成本 = 常数 |
| 可用资源 | 产能、人员出勤 | 资源 = 常数 |
| 收益率 | 投资回报、利率 | 收益率 = 常数 |

**实操建议**：找业务团队问：「你过去三年每个月这个数字是多少？」——如果数字一直在变，它就是不确认的。

### Step 2：问自己两个问题

1. **你知道参数的概率分布吗？** 有历史数据吗？能做分布拟合吗？还是只知道范围？
2. **你能接受多保守的解？** 你的业务愿意为「防御极端情况」多花多少钱？

### Step 3：选处理范式

| 问题 1 的答案 | 问题 2 的答案 | 选 |
|-------------|-------------|-----|
| 分布已知 | 期望最优 | 随机规划 |
| 范围已知 | 最坏可行 | 鲁棒优化 |
| 部分统计已知 | 两者之间 | 分布鲁棒 |

### Step 4：建模方式

**随机规划**：
1. 定义场景（离散化连续分布）
2. 写两阶段结构（这里决策 / 那里决策）
3. 建立对等模型（问题规模扩大 N 倍）

**鲁棒优化**：
1. 定义不确定性集合（箱式 / 椭球 / 预算式）
2. 推导对等模型（约束数量不变，结构变复杂）
3. 调节保守度参数（Γ, ε, ρ）

### Step 5：验证保守度

把结果和确定性模型对比——如果保守到不切实际：
- 调整不确定性集合（从箱式换成预算式）
- 放松机会约束（从 100% 可行到 95% 可行）
- 增加更多数据缩小不确定性边界

> 🆘 逃生通道：如果觉得上面的内容太多——先记住一个核心区别：**随机规划需要分布，鲁棒优化只需要范围。** 然后去读 [03 案例：新闻摊贩](03-case-newsvendor.md)，最直观地感受随机规划在做什么。
