## 2.1 两阶段随机规划

### 核心结构：「这里决策 → 观察 → 那里决策」

**生活类比**：双十一备货。第一阶段（8 月）：你提前两个月定原材料，不知道具体销量。第二阶段（10 月）：等到预售数据出来，你再决定每种产品具体生产多少。

**工作类比**：芯片制造。第一阶段：按预计需求建晶圆厂（投资数十亿美元）。第二阶段：等订单进来再调度具体产线生产（运营成本）。

### 数学形式（逐符号翻译）

```
min  cᵀx  +  E[Q(x, ξ)]
```

**符号含义**：
- `cᵀx`：第一阶段成本。x 是这里决策变量（采购量、产能建设），c 是单位成本。
- `E[Q(x, ξ)]`：第二阶段的期望成本。E 表示期望（也就是所有场景的加权平均）。
- `ξ` (读作 xi)：随机参数。你第二阶段才会知道它的实际值（比如需求）。
- `Q(x, ξ)`：看到随机参数后的第二阶段最优成本。对于给定的第一阶段决策 x 和实现的随机参数 ξ：
  ```
  Q(x, ξ) = min  qᵀy
            s.t. Wy ≥ h - Tx
                 y ≥ 0
  ```
  - `y`：第二阶段决策变量（产量、配送量）
  - `q`：第二阶段成本系数
  - `Wy ≥ h - Tx`：第二阶段约束。注意这里第二阶段决策 y 要弥补第一阶段决策的不完美——你备的料 x 不够时得紧急加单。

### 非预期约束（Non-anticipativity）

这是随机规划中最容易被忽略但最重要的概念。

**一句话人话**：第一阶段决策不能「看到未来」。你不能在知道需求是多少之后再决定备多少料。

**生活类比**：双十一 8 月备料时没人知道最终销量——如果你在 8 月就知道 10 月的数据，那就不是预测，是穿越。

**数学表达**：第一阶段决策 x 不能依赖于随机参数 ξ 的实现。也就是 x 对所有场景必须相同。这看起来简单，但新手经常犯错——在建模时无意中让 x 依赖场景。

**这里/那里的命名**：
- **这里以及现在（Here-and-Now）**：第一阶段——先做决策，不确定性还没揭晓
- **等一下再看（Wait-and-See）**：第二阶段——不确定性揭晓了，再做调整

## 2.2 场景生成

> 随机规划的核心操作之一：把连续分布变成一组离散的场景。

### 蒙特卡洛模拟（Monte Carlo）

**方法**：从已知分布中随机抽样，每个样本作为一个场景。

```
需求 ~ Normal(100, 20)
→ 抽取 N 个样本：D₁=95, D₂=112, D₃=87, ..., D_N=103
→ 每个场景概率 = 1/N（等权重）
```

**优缺点**：简单易用，但需要大量样本才能准确代表真实分布。标准差 σ 的估计误差 ~ 1/√N——要精度翻倍，样本量需要 4 倍。

### SAA（Sample Average Approximation）

**核心思路**：用抽样平均代替真实期望。

```
真实期望：E[f(x, ξ)] = ∫ f(x, ξ) · P(dξ)
SAA近似：  (1/N) · Σ f(x, ξₛ)
```

**关键定理**：随着样本量增加，SAA 的解以指数速度收敛到真实最优解。这是 SAA 能成立的理论保证。

**实际建议**：50-100 场景足够初步评估，200-500 场景用于正式决策。

### 矩匹配（Moment Matching）

**适用场景**：你知道目标分布的均值、方差，但不知道具体形状。

**方法**：生成一组场景 {ξ₁, ..., ξ_N}，使它们的样本均值 = μ、样本方差 = σ²。有一些数值优化方法可以做到，本课程不深入。

**什么时候用**：只知道历史均值方差、但不知道分布形态时。

### 情景树生成（Scenario Tree）

**适用场景**：多阶段决策（超过两个阶段）。每个阶段的决策依赖于之前阶段观察到的信息。

```
时间 t=1          t=2          t=3
  [决策] ──场景A── [决策] ──场景AA── [决策]
                 └──场景AB── [决策]
       └──场景B── [决策] ──场景BA── [决策]
                 └──场景BB── [决策]
```

**挑战**：树的大小随深度指数增长。深度 3 层 × 每层 5 个分支 = 125 个场景，深度 4 层 = 625 个场景。

### 重要性抽样（Importance Sampling）

**思路**：对极端场景施加更多权重，而不是均匀抽样。

**为什么重要**：与其均匀抽 10,000 个场景让大多数落在均值附近，不如重点抽 100 个「最危险」的场景（尾部事件），对它们给予更高权重。

## 2.3 对等模型（Deterministic Equivalent）

**生活类比**：把「万一明天下雨」的不确定性，转化为「我带一把伞」这个确定的决定。你对每种天气都有了一个确定的应对方案——伞、防晒霜、还是围巾。

**工作类比**：把「需求可能是 1000/1200/1500」的不确定性，转化为一组确定的约束和变量——「如果需求=1000，生产 y₁=...；如果需求=1200，生产 y₂=...；如果需求=1500，生产 y₃=...」——每个方案都是确定的。

### 核心思路

用一组场景 S = {1, 2, ..., N} 和概率 pₛ 代替连续随机变量，把随机问题变成一个巨大的确定性 LP。

**原始两阶段随机规划**：
```
min  cᵀx + Σ pₛ · qᵀyₛ
s.t. Tx + Wyₛ ≥ h,  ∀s
     x ≥ 0, yₛ ≥ 0, ∀s
```

**对等模型**（展开后）：
```
min  c₁x₁ + ... + cₙxₙ + p₁(q₁y₁₁ + ...) + p₂(q₁y₂₁ + ...) + ...
s.t. T₁₁x₁ + ... + W₁₁y₁₁ + ... ≥ h₁   (场景 1 约束)
     T₂₁x₁ + ... + W₂₁y₂₁ + ... ≥ h₂   (场景 2 约束)
     ...
     x ≥ 0, yₛ ≥ 0
```

### 规模膨胀

| 原问题 | N 场景对等模型 |
|--------|--------------|
| n 个变量 | ~ n + N×n' 个变量 |
| m 个约束 | ~ m + N×m' 个约束 |

**具体数字**：一个 100 变量、50 约束的 LP × 1000 场景
→ 对等模型：~100,100 变量、~50,050 约束
→ 完全可解（现代求解器几秒搞定）

一个 10,000 变量、5,000 约束的 LP × 100,000 场景
→ 对等模型：~10,010,000 变量、~5,005,000 约束
→ 需要分解方法

### 计算瓶颈与分解思想

**当 N 太大时**，直接解对等模型不可行。这时候需要：
- **Benders 分解**（也叫 L-shaped 方法）：把问题拆成主问题（第一阶段变量）和子问题（每个场景独立求解）
- **渐进式对冲算法**（Progressive Hedging）：并行求解每个场景问题，再通过协商调整第一阶段变量

## 2.4 鲁棒优化的不确定性集合

> 如果你不知道概率分布但知道范围——鲁棒优化就是你的工具。

### 三种核心集合

| 集合类型 | 形状 | 数学形式 | 保守度 | 计算代价 | 生活类比 |
|---------|------|---------|-------|---------|---------|
| 箱式（Box） | 超矩形 | `‖ŝ - ŝ₀‖∞ ≤ ρ` | 最高（最保守） | 最低 | 只知道在 0~100 之间 |
| 椭球式（Ellipsoidal） | 椭球 | `(ŝ - ŝ₀)ᵀΣ⁻¹(ŝ - ŝ₀) ≤ ρ²` | 中等 | 中等 | 知道大概在中心附近，越远可能性越低 |
| 预算式（Budget） | 多面体 | `Σᵢ ‖ŝᵢ - ŝ̂ᵢ‖/ŝ̂ᵢ ≤ Γ` | 可控 | 低 | 最多 K 个参数同时偏离 |

### 箱式（Box）

**生活类比**：你只知道明天温度在 15°C ~ 35°C 之间，内部分布完全不知道。

**数学含义**：每个不确定参数 aⱼ 独立地在 [āⱼ - âⱼ, āⱼ + âⱼ] 内取值。

**问题**：**过于保守**——因为它要求所有参数同时取最坏值。真实世界中，所有参数同时极端是几乎不可能的。箱式集是鲁棒优化的起点，但不是终点。

### 椭球式（Ellipsoidal）

**生活类比**：你不仅知道温度在 15°C ~ 35°C，还知道温度更可能靠近 25°C——离中心越远的温度越不可能。

**数学含义**：不确定性参数围绕中心点分布在一个椭球内，椭球的形状由协方差矩阵 Σ 决定。

**特点**：
- 比箱式更紧（不包含极端组合）
- 对等模型是 SOCP（二阶锥规划），可以用通用求解器解
- 需要知道协方差结构（数据要求比箱式高）

### 预算不确定性（Budget Uncertainty）—— 核心中的核心

**Bertsimas & Sim (2004) 的关键洞见**：并非所有不确定性参数会同时达到最坏值。

**生活类比**：你负责 10 个港口的进口。所有 10 个港口同时因为台风关闭的概率几乎为零——但 1-2 个港口同时出事的概率很大。

**数学模型**：引入预算参数 Γ（也叫 Budget of Uncertainty）：
- Γ = 0：退化为确定性模型（不相信任何不确定性）
- Γ = N：退化为箱式模型（所有参数可能最差）
- Γ = 5：最多 5 个参数可以同时取边界值

**一句话选择指南**：预算不确定性因为线性结构的保持而成为工业界最常用的鲁棒建模方式——对等模型仍然是 LP，不需要 SOCP 求解器。

## 2.5 线性优化中的鲁棒对等模型

> 不同不确定性集合对应的确定性等价形式。

### 箱式不确定性

**原始约束**：
```
Σ aⱼxⱼ ≤ b,  aⱼ ∈ [āⱼ - âⱼ, āⱼ + âⱼ]
```

**对等约束**（推导过程从略）：
```
Σ āⱼxⱼ + Σ âⱼ|xⱼ| ≤ b
```

**含义**：你把每个 aⱼ 取最坏值（对约束最不利的那个边界），然后要求加总不超限。注意 `|xⱼ|`：如果 xⱼ 可以是负的，取绝对值。

### 预算式不确定性

**原始约束**同上。

**对等约束**：
```
Σ āⱼxⱼ + max_{S∪{t}: |S|≤⌊Γ⌋, t∉S} [Σ_{j∈S} âⱼ|xⱼ| + (Γ - ⌊Γ⌋)âₜ|xₜ|] ≤ b
```

推导后等价为以下线性约束：
```
Σ āⱼxⱼ + Γ·z + Σ rⱼ ≤ b,  ∀j
z + rⱼ ≥ âⱼ|xⱼ|,  ∀j
z ≥ 0,  rⱼ ≥ 0,  ∀j
```

**关键洞见**：鲁棒对等模型和原问题是**同一类型**——LP → LP，SOCP → SOCP。这不会改变问题类别，这是鲁棒优化的工程魅力。

## 2.6 机会约束（Chance Constraint）的近似

> 生活类比：「90% 的情况下不能缺货」——你允许 10% 的缺货风险。

### 定义

```
P(g(x, ξ) ≤ 0) ≥ 1 - α
```

**逐符号翻译**：
- `P(...)`：概率
- `g(x, ξ) ≤ 0`：约束条件（比如「库存 ≥ 需求」）
- `1 - α`：置信水平。α = 0.05 意味着允许 5% 的违反概率

### 为什么机会约束难处理？

即使 ξ 是简单的分布（如正态分布），`P(g(x, ξ) ≤ 0) ≥ 1 - α` 定义的可行集通常是非凸的。非凸意味着你没法用现有的 LP/SOCP 求解器直接解。

### 三种近似方法

#### 1. 正态假设 + 解析近似

如果 ξ 服从正态分布 N(μ, σ²)，机会约束可以写成：

```
μ + z_α · σ ≤ 0
```

其中 z_α 是标准正态分布的 α-分位数（α=0.05 时 z₀.₀₅ ≈ 1.645）。

**含义**：你不只是考虑均值，还要在均值上加「1.645 倍标准差」的安全边际。

#### 2. 场景近似

用 N 个场景替换机会约束，要求约束在所有场景中成立：

```
g(x, ξₛ) ≤ 0,  ∀s = 1,...,N
```

**优缺点**：简单但可能过于保守（要求所有场景满足）或不够（如果 N 太小）。需要大量场景来保证覆盖 α 的违反概率。

#### 3. CVaR 近似 — 工业界最实用

**CVaR**（Conditional Value-at-Risk，条件风险价值）是「最坏 α 百分比」的期望值。

**生活类比**：VaR 是说「95% 的情况下亏损不超过 100 万」，CVaR 是说「如果亏损发生在那最差的 5%，平均会亏多少」。

**核心结果**：用 CVaR 约束代替机会约束——CVaR 约束是凸的、可解，且比机会约束更保守。数学上：

```
CVaR_α(g(x, ξ)) ≤ 0  →  P(g(x, ξ) ≤ 0) ≥ 1 - α
```

**为什么工业界爱用**：因为 CVaR 可以写成线性约束（当用场景近似时），而且提供的安全保证比机会约束更强。

> 🆘 记住两点：① 机会约束 = 允许一定概率违反 ② 把机会约束转换成 CVaR 约束是工业界最实用的做法。
